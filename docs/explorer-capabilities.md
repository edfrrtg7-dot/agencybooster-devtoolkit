# Explorer Capability Matrix

Internal reference for AgencyBooster DevToolkit observation coverage.
Generated by ABDT-012 audit.

## Runtime Explorer

| Capability | Status | Mechanism | Notes |
|------------|--------|-----------|-------|
| Extension startup | Supported | `Object.getOwnPropertyNames(window)` | One-time snapshot at start |
| Page navigation | Supported | `setInterval` polling (500ms) | Detects href changes; cross-origin resets content script |
| History (back/forward) | Supported | `popstate` event | Only user-initiated back/forward; pushState/replaceState NOT detected |
| Hash changes | Supported | `hashchange` event | Full coverage for hash-based routing |
| Visibility changes | Supported | `visibilitychange` event | Tab switch, minimize, fullscreen |
| pushState/replaceState | Not Supported | — | Would require monkey-patching `history.pushState`/`replaceState` |
| Tab lifecycle | Not Supported | — | Requires `chrome.tabs` API in background script |
| Service Worker navigation | Not Supported | — | Not visible from content script context |

**Health behavior:** Starts "healthy" (startup snapshot), transitions to "silent" after 30s without events. Returns to "healthy" on any new event.

## DOM Explorer

| Capability | Status | Mechanism | Notes |
|------------|--------|-----------|-------|
| Node additions | Supported | MutationObserver `childList` | Added nodes counted |
| Node removals | Supported | MutationObserver `childList` | Removed nodes counted |
| Attribute mutations | Supported | MutationObserver `attributes` + `attributeOldValue` | Captures attribute name and old value |
| Text mutations | Supported | MutationObserver `characterData` + `characterDataOldValue` | Captures old text value |
| Subtree observation | Supported | MutationObserver `subtree: true` | Entire document tree |
| Shadow DOM (nested) | Not Supported | — | Only observes roots under `<html>` |
| iframe content | Not Supported | — | Cross-origin iframes are separate documents |
| High-mutation pages | Caution | — | SPA frameworks produce many observations; storage grows quickly |

**Health behavior:** "healthy" if mutations observed, "idle" if page is completely static.

## Storage Explorer

| Capability | Status | Mechanism | Notes |
|------------|--------|-----------|-------|
| localStorage.setItem | Supported | Prototype patch | Captures key, old value, new value |
| localStorage.removeItem | Supported | Prototype patch | Captures key, old value |
| sessionStorage.setItem | Supported | Prototype patch | Same mechanism |
| sessionStorage.removeItem | Supported | Prototype patch | Same mechanism |
| Storage.clear() | Not Supported | — | Would require enumerating all keys before clearing |
| Cross-tab changes | Not Supported | — | Prototype patch only catches current page writes |
| Cross-origin storage | Not Supported | — | Different storage contexts |
| Storage events | Not Used | — | `storage` event fires after write; our patch fires before (more reliable) |

**Health behavior:** "idle" if page doesn't use Storage APIs. This is correct — the explorer is running but has nothing to observe.

## Network Explorer

| Capability | Status | Mechanism | Notes |
|------------|--------|-----------|-------|
| fetch() | Supported | `window.fetch` patch | URL, method, status, duration |
| XMLHttpRequest | Supported | `open`/`send` patches | URL, method, status, duration |
| Failed requests | Supported | try/catch in fetch, XHR loadend | Status omitted on network error |
| Request bodies | Not Supported | — | Would require reading ReadableStream (single-use) |
| Response bodies | Not Supported | — | Would require `tee()` on Response body |
| Request headers | Not Supported | — | Not captured |
| Response headers | Not Supported | — | Not captured |
| WebSocket | Not Supported | — | Different protocol, not interceptable via prototype |
| Server-Sent Events | Not Supported | — | EventSource not patched |
| Beacon API | Not Supported | — | `navigator.sendBeacon` not patched |
| Navigation requests | Not Supported | — | Occur before content script injection |
| Resource loading | Not Supported | — | Browser-initiated (images, scripts, etc.) |

**Health behavior:** "idle" if page doesn't make fetch/XHR calls after injection. This is correct.

## Diagnostics Accuracy

| Component | What It Shows | Accuracy |
|-----------|---------------|----------|
| Explorer health dots | healthy/idle/silent/error | Correct — matches runtime state |
| Observation counts | Total per explorer | Correct — incremented on every `record()` |
| Rolling rate | obs/sec over 10s window | Correct — pruned on each poll |
| Live viewer summaries | Per-observation text | Accurate for DOM/Storage/Network; Runtime events now show trigger-specific summaries (ABDT-012 fix) |
| Clipboard exports | Full diagnostics text | Accurate — includes all health stats |

## Health State Definitions

| State | Condition | Meaning |
|-------|-----------|---------|
| healthy | `running && totalObservations > 0 && lastActivity < 30s` | Active — recent observations |
| idle | `running && totalObservations === 0` | Running but no events observed |
| silent | `running && totalObservations > 0 && lastActivity > 30s` | Had events, none recently |
| error | `!running` | Explorer failed to start or was stopped |

## MV3 Constraints Summary

The extension uses only `"activeTab"` and `"scripting"` permissions. All explorers run in the content script context. This means:

- No `webRequest` blocking or observation (read-only in MV3 anyway)
- No background-level network observation
- No access to `chrome.tabs` from content script
- No cross-origin iframe access
- Content script is injected after page load — pre-injection events are missed
- Page navigation resets the content script (new session on new page)

These are intentional trade-offs for minimal permissions.
